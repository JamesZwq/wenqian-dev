# 🧪 快速测试指南

## 本地测试 P2P 聊天

### 方法 1：使用两个浏览器窗口

1. **启动开发服务器**
   ```bash
   npm run dev
   ```

2. **打开第一个窗口**
   - 访问 `http://localhost:3000`
   - 点击 `[ P2P_CHAT ]` 按钮
   - 复制显示的 Peer ID（例如：`abc123xyz`）

3. **打开第二个窗口**（隐身模式或不同浏览器）
   - 访问 `http://localhost:3000/chat`
   - 复制第二个窗口的 Peer ID
   - 在 "Connect to Peer" 输入框中粘贴第一个窗口的 ID
   - 点击 "Connect"

4. **开始聊天**
   - 连接成功后会看到绿色的 "Connected" 指示器
   - 同时会有 50 个青绿色粒子爆发效果
   - 在任一窗口输入消息并发送
   - 另一个窗口会实时收到消息

### 方法 2：使用两台设备

1. **确保两台设备在同一网络**

2. **在主机上启动服务器**
   ```bash
   npm run dev
   ```

3. **获取主机 IP 地址**
   ```bash
   # macOS/Linux
   ifconfig | grep "inet "
   
   # Windows
   ipconfig
   ```

4. **在两台设备上访问**
   - 设备 A：`http://[主机IP]:3000/chat`
   - 设备 B：`http://[主机IP]:3000/chat`

5. **交换 Peer ID 并连接**

## 常见问题排查

### 问题：Peer ID 一直显示 "Generating..."
**原因**：PeerJS 服务器连接失败  
**解决**：
- 检查网络连接
- 等待几秒钟
- 刷新页面重试

### 问题：点击 Connect 后没有反应
**原因**：对方的 Peer ID 不正确或对方不在线  
**解决**：
- 确认 Peer ID 完全正确（区分大小写）
- 确保对方的页面仍然打开
- 尝试反向连接（对方连接你）

### 问题：消息发送失败
**原因**：连接已断开  
**解决**：
- 检查连接状态指示器是否为绿色
- 刷新页面重新建立连接

### 问题：粒子效果不显示
**原因**：Canvas 渲染问题  
**解决**：
- 检查浏览器控制台是否有错误
- 尝试使用 Chrome/Edge 浏览器
- 检查硬件加速是否开启

## 性能测试

### 消息延迟
- 本地网络：< 50ms
- 同城网络：< 100ms
- 跨城网络：< 200ms

### 粒子性能
- 连接粒子：50 个，持续 1 秒
- 消息粒子：20 个，持续 1 秒
- 帧率：应保持 60 FPS

### 内存使用
- 初始加载：~50MB
- 100 条消息：~60MB
- 1000 条消息：~100MB
