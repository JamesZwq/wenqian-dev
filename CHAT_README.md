# P2P Chat Feature

## 功能特点

- ✨ **点对点加密通信**：使用 WebRTC 技术实现真正的 P2P 连接
- 🎨 **极致视觉设计**：渐变背景、粒子特效、流畅动画
- 🚀 **实时消息传输**：无需服务器中转，直接端到端通信
- 📱 **响应式设计**：完美支持桌面和移动设备
- 🎭 **动态粒子系统**：连接和消息发送时的视觉反馈

## 使用方法

### 1. 访问聊天页面

在首页点击 `[ P2P_CHAT ]` 按钮，或直接访问 `/chat` 路由。

### 2. 获取你的 Peer ID

页面加载后会自动生成一个唯一的 Peer ID，显示在 "Your Peer ID" 输入框中。

### 3. 连接到对方

有两种方式建立连接：

**方式 A：你主动连接**
1. 将你的 Peer ID 发送给对方（通过其他通讯方式）
2. 对方将他们的 Peer ID 发送给你
3. 在 "Connect to Peer" 输入框中输入对方的 Peer ID
4. 点击 "Connect" 按钮

**方式 B：等待对方连接**
1. 将你的 Peer ID 发送给对方
2. 等待对方输入你的 ID 并连接
3. 连接建立后会自动显示聊天界面

### 4. 开始聊天

连接成功后：
- 在底部输入框输入消息
- 按 Enter 或点击 "Send" 按钮发送
- 你的消息显示在右侧（紫粉渐变）
- 对方的消息显示在左侧（青色边框）

## 技术实现

### 核心技术栈

- **PeerJS**：WebRTC 的简化封装，处理 P2P 连接
- **Framer Motion**：流畅的动画和过渡效果
- **Canvas API**：粒子特效渲染
- **Next.js 16**：React 框架和路由

### 设计亮点

1. **渐变背景**：深空紫到粉色的多层渐变
2. **网格动画**：无限循环的背景网格移动
3. **粒子系统**：
   - 连接成功时爆发 50 个粒子（青绿色）
   - 每次发送消息产生 20 个粒子（紫色）
   - 粒子具有物理运动和生命周期
4. **消息动画**：每条消息以缩放+淡入效果出现
5. **状态反馈**：连接状态的实时视觉指示

### 字体选择

- **标题**：Orbitron（科技感未来风格）
- **正文**：等宽字体（代码风格）

## 安全性说明

- 所有消息通过 WebRTC 的加密通道传输
- 不经过任何中间服务器（除了初始的信令服务器）
- Peer ID 是临时的，每次刷新页面会重新生成
- 建议在可信网络环境下使用

## 浏览器兼容性

支持所有现代浏览器：
- Chrome/Edge 80+
- Firefox 75+
- Safari 14+
- Opera 67+

## 故障排除

**无法连接？**
- 检查双方的 Peer ID 是否正确
- 确保网络环境允许 WebRTC 连接
- 尝试刷新页面重新生成 Peer ID

**消息发送失败？**
- 检查连接状态指示器是否为绿色
- 确保对方的页面仍然打开
- 尝试重新建立连接
